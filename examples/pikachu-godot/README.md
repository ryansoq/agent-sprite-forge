# Pikachu Mini RPG

A small but mostly-complete Godot 4 RPG demo: title screen, larger overworld
with camera follow and obstacles, four-direction walking, multi-monster
encounters, turn-based battle with a move/bag/party menu, capturing wild
monsters into a party of up to six, healing, items, save/load, and a pause
menu.

The art is **placeholder pixel art** drawn programmatically with Pillow.
Claude Code does not have the `image_gen` tool that this repository's skills
rely on, so the sprites are hand-coded shapes rather than image-generated. To
upgrade to real assets, run `$generate2dsprite` and `$generate2dmap` inside
Codex and drop the resulting PNGs into `assets/`.

> Personal/educational demo only. Pikachu is owned by Nintendo / The Pokémon
> Company — the player sprite is a placeholder lookalike. Do not redistribute
> or use commercially. The wild monsters (Volty, Twigling, Embertail) are
> original creatures.

## Requirements

- Godot 4.2 or newer (tested on 4.4.1)
- Python 3 + Pillow (only if you want to regenerate the placeholder PNGs)

## Open and play

```bash
godot4 --path examples/pikachu-godot
```

Or open the Godot project manager and import `project.godot`.

## Controls

- **Arrow keys** — move
- **Space / Enter** — confirm (title screen)
- **R** — reset save (title screen)
- **ESC** — pause / back

## Features

### Overworld

- 1280×720 world with camera follow and tree/rock walls.
- Five regions of tall grass, scattered around routes.
- Walking into tall grass has an 18% encounter chance per entry, with a
  cooldown so re-entering after a battle does not insta-trigger.
- A pink **healing pad** in the center fully restores party HP and grants a
  Potion each time you step on it.
- HUD shows active monster HP, item counts, capture total.

### Battle

- Turn-based with a state-driven menu: **Fight / Bag / Party / Run**.
- Active monster's moves come from `MonsterData`. Pikachu starts with
  Thunderbolt, Quick Attack, Thunder Wave (paralyze), and Recover.
- Three wild monster types with their own move sets:
  Volty (electric), Twigling (grass), Embertail (fire).
- HP bars (visual) plus damage flash and side-shake on hit.
- Status: `paralyze` gives a 50% skip chance per enemy turn.
- Items: **Potion** heals 20 HP. **Pokeball** captures wild monsters with
  chance scaling from low HP (lower → higher catch chance).
- Switching: choose any non-fainted party member from the Party menu.
  When the active monster faints, you are forced to choose a replacement.
- Whiteout: if every party member faints, the game heals you fully and
  rushes you back to the center of the overworld.

### Party

- Up to 6 monsters. Wild captures are appended.
- HP persists between battles and is saved.

### Save / load

- A save file is written to `user://pikachu_save.cfg` after every battle
  result, every healing, and on pause-menu Save.
- Title screen shows the save summary and offers Continue or Reset (R).

### Pause menu (ESC in overworld)

- Resume / Save / Save & Title.

## Regenerate placeholder assets

```bash
python3 tools/make_assets.py
```

Pillow only — no AI generation. Outputs to `assets/`.

## Replacing placeholders with real sprites

In Codex, run:

```text
Use $generate2dsprite to create a 4x4 player_sheet for a pikachu-style
electric mouse walking in four directions, retro JRPG pixel-art style,
32x32 cell.
```

Drop the transparent frames over `assets/pikachu_down.png`,
`pikachu_up.png`, `pikachu_side.png` (or wire an `AnimatedSprite2D` for a
full walk cycle). For the map:

```text
Use $generate2dmap with engine_target=Godot_TileMap to produce a small
grass meadow with tall-grass encounter zones, trees, rocks, water pond,
and a healing pad, 1280x720 logical pixels, 32-pixel tiles.
```

That gives you a real `TileMapLayer` you can swap for the
programmatically-spawned trees and grass in `Overworld.gd`.

## Layout

```
examples/pikachu-godot/
  project.godot
  assets/                   # placeholder PNGs (generated by tools)
  scenes/
    TitleScreen.tscn        # title + save summary
    Overworld.tscn          # 1280x720 world; trees/grass spawned in script
    Player.tscn             # CharacterBody2D + Camera2D + 4-dir sprite
    Battle.tscn             # state-driven battle UI (4 menus)
    PauseMenu.tscn          # overlay CanvasLayer
  scripts/
    MonsterData.gd          # autoload: monster definitions + WILD_POOL
    MoveData.gd             # autoload: move definitions (damage/heal/status)
    Fade.gd                 # autoload: black-screen fade between scenes
    GameState.gd            # autoload: party, inventory, save/load
    TitleScreen.gd
    Player.gd               # 4-dir movement + walk bobbing
    Overworld.gd            # spawn world, encounters, healing pad, pause
    Battle.gd               # full battle state machine + animations
    PauseMenu.gd
  tools/make_assets.py      # Pillow placeholder generator
```

## Limitations / what's not in scope

- No XP / leveling / move learning. Stats are flat.
- No type effectiveness (electric vs. water etc.) — flat damage rolls.
- No NPC dialogue, no shops, no multi-zone overworld.
- No audio. (Sound would need real audio files; out of scope here.)
- One animation frame per direction. Walk uses a position bob, not a
  per-frame sheet; swap to `AnimatedSprite2D` once Codex generates real
  walk frames.
- Overworld trees/rocks/grass are programmatically spawned in
  `Overworld.gd`, not a `TileMapLayer`. Easier to replace with a real
  tilemap from `$generate2dmap` later.

The point of this demo is to give you a playable scaffold in one click and
a clear set of replacement points for Codex-generated assets.
